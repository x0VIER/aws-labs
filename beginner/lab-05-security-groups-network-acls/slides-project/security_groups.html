Step 1: Understanding Security Groups

Instance-level firewalls in AWS

Security Groups act as virtual firewalls for your EC2 instances to control inbound and outbound traffic.

Key Characteristics:

Stateful: Return traffic is automatically allowed, regardless of inbound rules
Allow Rules Only: You can only specify allow rules, not deny rules
Instance Level: Associated with instances, not subnets
Evaluated Together: All rules are evaluated before deciding to allow traffic

Default Configuration:

  • All outbound traffic is allowed
  • No inbound traffic is allowed
  • Can be associated with multiple instances
  • Instances can have multiple security groups

Common Security Group Rules:

Type Port Source/Destination Purpose
SSH 22 Your IP Secure shell access
HTTP 80 0.0.0.0/0 Web access
HTTPS 443 0.0.0.0/0 Secure web access
