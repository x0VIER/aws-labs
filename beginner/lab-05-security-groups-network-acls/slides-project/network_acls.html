Step 3: Understanding Network ACLs

Subnet-level firewalls in AWS

Network Access Control Lists (NACLs) are an optional layer of security that acts as a firewall for controlling traffic in and out of a subnet.

Key Characteristics:

Stateless: Return traffic must be explicitly allowed by rules
Allow and Deny Rules: You can specify both allow and deny rules
Subnet Level: Associated with subnets, not instances
Rule Number Priority: Rules are evaluated in order from lowest to highest

Default Configuration:

  • All inbound and outbound traffic is allowed
  • Each subnet is associated with a default NACL
  • A NACL can be associated with multiple subnets
  • A subnet can only be associated with one NACL

Example NACL Rules:

Rule # Type Protocol Port Range Allow/Deny
100 HTTP TCP 80 ALLOW
110 HTTPS TCP 443 ALLOW
120 SSH TCP 22 DENY
