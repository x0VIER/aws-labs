Step 4: Configure a Network ACL

Setting up subnet-level security

1

In the VPC console, navigate to Network ACLs in the left sidebar.

2

Click Create network ACL.

3

Enter a Name (e.g., Web-Server-NACL) and select your VPC.

4

Click Create network ACL.

5

Select your new NACL and click the Inbound rules tab.

6

Click Edit inbound rules and add rules for HTTP (80), HTTPS (443), and SSH (22).

7

Click the Outbound rules tab, then Edit outbound rules.

8

Add a rule to allow all outbound traffic (0.0.0.0/0).

9

Under Subnet associations, associate the NACL with your subnet.

AWS CLI Alternative:

# Create a new Network ACL
aws ec2 create-network-acl \
  --vpc-id vpc-1234567890abcdef0 \
  --tag-specifications 'ResourceType=network-acl,Tags=[{Key=Name,Value=Web-Server-NACL}]'

# Add an inbound rule for HTTP traffic
aws ec2 create-network-acl-entry \
  --network-acl-id acl-1234567890abcdef0 \
  --ingress \
  --rule-number 100 \
  --protocol tcp \
  --port-range From=80,To=80 \
  --cidr-block 0.0.0.0/0 \
  --rule-action allow
