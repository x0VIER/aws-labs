Step 4: Configure a Lifecycle Policy

Automating object transitions and expirations

Lifecycle policies automatically transition objects between storage classes or delete them based on defined rules.

1

In your bucket, go to the Management tab.

2

Click Create lifecycle rule.

3

Enter a Rule name (e.g., archive-old-files).

4

Define the rule scope (apply to all objects or filter by prefix/tags).

5

Configure transitions (e.g., move to Glacier after 90 days).

6

Set expiration rules if needed (e.g., delete after 365 days).

7

Review and click Create rule.

Common Storage Class Transitions:

Standard → Standard-IA: For data accessed less than once a month
Standard-IA → Glacier: For long-term archival with occasional access
Any Class → Glacier Deep Archive: For data that rarely needs to be accessed
