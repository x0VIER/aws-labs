Step 5: Understand IAM Roles

Granting permissions to AWS services

IAM roles are similar to users but are intended to be assumable by anyone who needs them, including AWS services.

Key Differences from Users:

  • Not associated with one specific person
  • Can be assumed temporarily
  • No long-term credentials

Common Use Case: Allow EC2 instances to access S3 buckets without storing AWS credentials.

1

In the IAM console, go to Roles and click Create role.

2

For trusted entity type, select AWS service and choose EC2.

3

Attach the AmazonS3ReadOnlyAccess policy and name the role.

Role Trust Relationship:

{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": {
      "Service": "ec2.amazonaws.com"
    },
    "Action": "sts:AssumeRole"
  }]
}
