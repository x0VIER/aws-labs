Step 2: Create IAM Policies

Defining custom permissions

While AWS managed policies are useful, you often need to create custom policies for specific permissions.

1

In the IAM console, go to Policies and click Create policy.

2

Select the JSON tab to write your policy document.

3

Add tags (optional), review, and name your policy.

4

Click Create policy to save it.

This example policy allows users to manage only their own access keys.

Example Policy: ManageOwnAccessKeys

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "iam:CreateAccessKey",
        "iam:DeleteAccessKey",
        "iam:ListAccessKeys",
        "iam:UpdateAccessKey"
      ],
      "Resource": "arn:aws:iam::*:user/${aws:username}"
    }
  ]
}
