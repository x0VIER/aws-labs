Step 4: Test User Permissions

Verifying that your IAM configuration works

After creating users and assigning them to groups, verify that the permissions work as expected:

1

Sign out of your admin user account.

2

Sign in as the s3-dev-1 user.

3

Try to access the S3 console. You should have full access.

4

Try to access the EC2 console. You should see an access denied message.

5

Repeat with ec2-support-1 user to verify EC2 read-only access.

✅ Allowed: s3-dev-1 user can manage S3 resources

⚠️ Limited: ec2-support-1 user can only view EC2

❌ Denied: Cross-service access is blocked
